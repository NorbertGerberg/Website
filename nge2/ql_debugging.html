Debugging

Some information to help you debug your application.


Debug.hpp

Debug.hpp inside of Core includes macros for breaking and asserting your program with NGE_ASSERT(vl) and NGE_DEBUGBREAK.


VersionInfo.hpp

VersionInfo.hpp inside of Core includes a macro that returns the current engine version as an uint16 NGE_ENGINE_INFO_VERSION.
You are also able to get the compiler name and version with the macros defined in this header. This is very important for the World library as it relies on the compiler specific typeid names.
So a game world file created using the MSVC compiler can not be opened with the GCC compiler.


Memory Leaks

At no point does the engine cause intentional memory leaks. The engine auto-checks for any leaks when using eEntry.
I can also recommend Dr. Memory to check for any potential leaks inside your program.
